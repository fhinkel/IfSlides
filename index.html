Git: Version control system

  • Git allows you to track your changes and collaborate with others
  • Install git http://git-scm.com/downloads
  • distributed (everybody has a local copy)
  • origin refers to a remote repository that everbody shares, e.g., GitHub
  • Git automatically merges changes from several people
  • Branches allow you to work on several versions in parallel, for example for features or bug fixes
  • Good tutorial: http://nyuccl.org/pages/GitTutorial/

Git origin and upstream

  • You can specify several remotes
    • to see all remote repositories: 
      git remote
    • to add a new remote repository that you call _name_ 
      git remote add _name_ _url_
      and to delete a remote repository 
      git remote delete _name_
    • sync your repository with changes from upstream: 
      git fetch upstream
      git rebase upstream/master
      (rebase is like merge but adding the changes in the order they were made)

Kanban Board

  • From Japan, used in automobile industry, ~1970
  • Write down tasks you think are necessary for the project
  • Pull principle: you "pull" what you want to work on, nobody "pushes" it on you
  • WIP - Work in progress limited to 2 tasks
  • Put an avatar on the task(s) that you are working on
  • Update the board as you work or during StandUp, you can also add/remove new tasks
  • Make 2 avatars (your name works best), both avatars should be the same
  • as a pair, pick your first task, move it to work in progress, stick your avatars on top
  • if it is a knowledge task, document it on the Wiki on GitHub
  • In the directory, where you checked out your forked repository, add my repository as upstream: 
    git remote add upstream git@github.com:fhinkel/SimpleChatServer.git 
    git remote --verbose
    origin	git@github.com:YOURUSERNAME/SimpleChatServer.git (fetch)
origin	git@github.com:YOURUSERNAME/SimpleChatServer.git (push)
upstream	git@github.com:fhinkel/SimpleChatServer.git (fetch)
upstream	git@github.com:fhinkel/SimpleChatServer.git (push)
  • Make sure your repository is up to date with upstream 
    git checkout master
                    git fetch upstream
                    git rebase upstream/master
                    git push origin master
  • Create and checkout a new branch called exercise3 
    git checkout -b exercise2
  • push your changes to your remote repository: 
    git push origin exercise3
    You can check on GitHub, that you have a branch master and a branch exercise3
